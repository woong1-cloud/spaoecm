# 재고 대시보드 (엑셀 → 웹)

엑셀을 업로드하면 스냅샷(날짜별 재고)을 로컬 DB에 저장하고, 웹에서 다음을 확인할 수 있습니다.

- 📊 **상품분석판매** - 판매량 기반 재고 분석
- 📦 **물류재고 통합** - 물류창고 재고 자동 매칭
- 🔥 **위험 SKU 써머리** - 판매 높은데 재고 없는 상품 우선 표시
- 🎯 **15자리 SKU 매칭** - 두 엑셀 데이터 자동 연결
- 📈 **발주 제안** - 7일 판매량 기반 소진 예상

## 1) 설치

```bash
python -m venv .venv
.\.venv\Scripts\activate
pip install -r requirements.txt
```

## 2) 실행

### Flask 웹 대시보드

```bash
python dashboard_v2.py
```

브라우저에서 확인:
- `http://127.0.0.1:5003`
- 기본 비밀번호: `1234`
- 로그인 후 비밀번호 변경 가능

### 엑셀 업로드 방법

**한 화면에서 최대 3개 파일을 동시에 업로드:**

1. **상품분석판매 (필수)** - CSV 파일
   - 판매수량, 현재재고 데이터
   - SKU 자동 생성 (15자리)
   - 예: `20260129_091301_ProductPrdchart.csv`

2. **물류센터1 재고 (선택)** - Excel 파일
   - 솔리드+아소트 가용재고
   - SKU 매칭으로 자동 업데이트
   - 예: `test0129_center1.xlsx`

3. **물류센터2 재고 (선택)** - Excel 파일
   - 솔리드+아소트 가용재고
   - SKU 매칭으로 자동 업데이트
   - 예: `test0129_center2.xlsx`

**📤 버튼 하나로 한 번에 처리됩니다!**

**참고:**
- 물류재고 = 센터1 + 센터2 (자동 합계)
- 실가용재고 = 현재재고 + 물류재고

### Streamlit 버전 (로컬 / Streamlit Cloud 배포)

streamlit run app.py

## 3) 엑셀 데이터 형식

### 상품분석판매 (CSV)

필수 컬럼:
- D열: `상품명` (SKU 베이스 10자리 포함)
- E열: `옵션` (컬러/사이즈 코드)
- G열: `재고` (현재 재고)
- J열: `판매수량` (7일치)

### 물류재고 (Excel)

필수 컬럼:
- F열: `상품` (SKU 15자리)
- P열: `솔리드가용재고`
- Q열: `아소트가용재고`

## 4) 엑셀 컬럼(권장)

필수:
- `sku` (또는 `상품코드`/`품목코드`)
- `stock` (또는 `재고`/`현재재고`)

권장:
- `name`(상품명), `category`(카테고리)
- `min_stock`(최소재고)
- `lead_time_days`(리드타임 일), `safety_stock`(안전재고)

## 4) 일자별 변동을 보려면

같은 SKU에 대해 **다른 날짜**로 스냅샷을 여러 번 업로드하면, “품목별 일자 재고 변동” 차트가 활성화됩니다.

## 5) 백업 및 복원

### 백업 방법

대시보드에서 "백업/내보내기" 버튼 또는 "백업" 메뉴 클릭:

**1. 엑셀 내보내기**
- 최신 스냅샷 데이터를 Excel로 다운로드
- 파일명: `재고현황_2026-01-29.xlsx`
- 용도: 외부 분석, 공유, 보고서

**2. 전체 DB 백업**
- 모든 히스토리 포함
- 파일명: `inventory_backup_20260129_153045.db`
- 용도: 완전한 백업, 복원용

### 복원 방법

**DB 복원:**
1. 서버 중지 (Ctrl+C)
2. 현재 `inventory.db` 백업
3. 복원할 `.db` 파일을 `inventory.db`로 이름 변경
4. 서버 재시작

**권장 백업 주기:**
- 중요 데이터 업로드 후 즉시
- 정기적으로 주 1회
- 대량 수정 전 반드시

## 6) 데이터 저장 위치

업로드된 데이터는 프로젝트 폴더의 `inventory.db`(SQLite)에 저장됩니다.

- **/test** 경로 비노출 (외부에서 서버 탐지 방지)

- **debug** 자동 해제**실행 방법 (Windows):**

```cmd
set DEPLOY_MODE=1
python dashboard_v2.py
```**실행 방법 (Linux/Mac):**

```bash
DEPLOY_MODE=1 python dashboard_v2.py
```
